# Security Policy

## Supported Versions

Security fixes are currently targeted at the latest published `0.1.x` release line.

## Reporting a Vulnerability

Please avoid opening public issues for security-sensitive reports.

Preferred path:

1. Use GitHub's private vulnerability reporting for `whitebalsamic/invoice-router` if it is enabled.
2. If private reporting is unavailable, contact the maintainer through GitHub and request a private follow-up channel.

When reporting an issue, please include:

- affected version or commit
- reproduction steps
- impact assessment
- any suggested mitigation

You can expect an acknowledgement as soon as practical, followed by triage, remediation, and coordinated disclosure guidance where appropriate.
